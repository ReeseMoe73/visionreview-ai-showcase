# VisionReview AI — Demo Recording Plan

## Recording Objective

Record a polished screen walkthrough that demonstrates VisionReview AI as a working local beta product.

The demo should look professional, focused, and credible for:

- LinkedIn
- GitHub public showcase
- Portfolio
- Job interviews
- SkreenKings AI product validation

---

## Recommended Recording Setup

### Screen Recording

Use one of:

```text
OBS Studio
Loom
Screen Studio
Camtasia
Windows Snipping Tool video capture
```

OBS is recommended if you want higher control.

### Browser Zoom

Use:

```text
90% or 100%
```

If the right-side panel feels crowded, use 90%.

### Screen Size

Recommended:

```text
1920 × 1080
```

or at least:

```text
1280 × 720
```

### Cursor

Keep cursor visible.

Move slowly and intentionally.

---

## Pre-Recording Checklist

Before recording:

- [ ] Backend is running.
- [ ] Frontend is running.
- [ ] Report page loads.
- [ ] Test video report is ready.
- [ ] Segment tags are already applied.
- [ ] At least one segment is Favorite or Keep.
- [ ] Client metadata is filled in.
- [ ] Metadata is saved.
- [ ] Last saved timestamp is visible.
- [ ] Client HTML opens correctly.
- [ ] Client ZIP downloads correctly.
- [ ] No private data is visible.
- [ ] Browser tabs are clean.
- [ ] Desktop is clean.
- [ ] Notifications are muted.

---

## Suggested Demo Video Flow

### Scene 1 — Product Overview

Show:

```text
Report page
Video player
Segment list
```

Do not spend too long here.

### Scene 2 — Review Mode

Show:

```text
Review status
Tags
Notes
Bulk Tags
```

Keep this section concise.

### Scene 3 — Reel Builder

Show:

```text
Timeline
Compare
QC
```

Avoid clicking through every tab. Pick the strongest sections.

### Scene 4 — Client Handoff

Show:

```text
Client Metadata
Save Metadata
Last saved
Client Review HTML
Download Client Package ZIP
Send-Ready Summary
```

This is the strongest commercial feature. Spend the most time here.

---

## What Not to Show

Do not show:

- API keys
- `.env` files
- private GitHub repo settings
- database files
- raw client data
- personal email inbox
- terminal errors
- OneDrive file explorer with unrelated private folders
- long backend logs
- implementation source code
